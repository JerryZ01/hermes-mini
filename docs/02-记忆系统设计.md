# 第二章 · 记忆系统：让 Agent 记住你

> 本章配套代码：`src/memory/`

---

## 本章目标

- 理解 Working Memory（工作记忆）和 Long-term Memory（长期记忆）的区别
- 实现 FTS5 全文搜索 + LLM 摘要的跨会话检索
- 实现 Honcho 用户画像：记住用户偏好、风格、习惯
- 理解为什么 Hermes 把"记忆"放在核心位置

---

## 为什么记忆是 Agent 的分水岭？

一个没有记忆的 Agent = 每次对话都是陌生人。

一个有记忆的 Agent = 越来越懂你，越用越顺手。

Hermes 的核心卖点之一就是**内置的学习闭环**——
它不只记住你说过什么，还**主动提炼你的特征**（Honcho），在合适的时机调用相关记忆。

---

## 三层记忆架构

```
┌─────────────────────────────────────┐
│  Honcho 用户画像层（最顶层）          │ ← 提炼用户特征，主动注入上下文
├─────────────────────────────────────┤
│  Long-term Memory 长期记忆层         │ ← SQLite FTS5 + LLM 摘要，跨会话
├─────────────────────────────────────┤
│  Working Memory 工作记忆层（最底层）   │ ← 当前会话的上下文窗口
└─────────────────────────────────────┘
```

---

## Working Memory

当前会话的上下文，通常是最近 N 轮对话。

限制：token 上限（GLM-5 ~128K context，Qwen ~32K）

策略：达到上限时做**上下文压缩**（下一章会讲到）

---

## Long-term Memory（重点）

问题：Agent 历史对话太多，检索困难。

方案：
1. 每轮对话结束后，把关键内容存入 SQLite（FTS5 全文索引）
2. 新会话开始时，用 LLM 提炼"当前任务相关的历史记忆"
3. 相关记忆作为 system prompt 注入上下文

---

## Honcho 用户画像

这是 Hermes 的独特设计：不是记录你说的话，而是**记录你是什么样的人**。

举例：
- "杰哥偏好简洁的回答，不喜欢废话"
- "杰哥习惯用中文，不喜欢中英混杂"
- "杰哥对 Agent 编程很感兴趣"

这些画像在每次对话中**主动注入**，让 Agent 的风格越来越贴合用户。
