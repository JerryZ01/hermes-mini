> 本章配套代码：`src/tools/`

---

## 本章目标

- 理解 Tool Use 在 Agent 架构中的定位
- 实现通用的工具注册与动态调用系统
- 理解工具结果的注入方式（Tool Result → Message）
- 内置工具集：文件读写、Shell 命令、网页搜索、计算器

---

## 工具是 Agent 的手脚

没有工具的 Agent 只能"说话"，不能改变外部世界。

有了工具之后，它可以：
- 读取文件、分析代码
- 执行 Shell 命令，运行测试
- 搜索网页，获取最新信息
- 读写数据库，操作 API

这就是"Agent"和"Chatbot"的本质区别：**Agent 可以影响环境，Chatbot 只能响应文字。**

---

## 工具调用的完整链路

```
用户输入："帮我查一下今天的天气"
     ↓
Agent Loop 推理：需要调用 weather 工具
     ↓
【工具调用】{"name": "weather", "args": {"city": "上海"}}
     ↓
【工具执行】调用外部 API，拿到结果
     ↓
【结果注入】Tool Result 作为消息注入上下文
     ↓
Agent Loop 继续推理：已有天气数据，生成回复
     ↓
输出："今天上海晴，25度……"
```

---

## 工具定义格式

每个工具是一个 Python 函数 + 装饰器元数据：

```python
@register_tool(name="天气查询", description="查询指定城市的实时天气")
def weather(city: str) -> str:
    """查询城市天气"""
    # 实际调用逻辑
    return f"{city}今天晴，25度"
```
